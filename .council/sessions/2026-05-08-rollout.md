# Council session: Quiet Ledger rollout + structure + weekly scrape

## Why convened

The Chairman has approved Quiet Ledger and opened the Council. The
PR (#17) lands the architectural scaffold, the spike, and the
foundational design tokens. Three streams of follow-on work are now
sequenced.

## Scope

Three phases, in order. Each phase is its own branch and PR.

### Phase A — Quiet Ledger production application

Apply Quiet Ledger to the rest of the production site. The
foundational `.dp-*` retune already shipped on PR #17, so card
review pages, bank hubs, airline programme pages, salary-transfer
slug pages, and valuations inherit the new palette and component
rules for free. The remaining work:

- **`src/pages/index.astro` — full rewrite** to the 4-section spec
  (masthead-dateline + 5-story typographic index → live module
  combining tracker + deals → latest analysis feed → newsletter +
  RSS-by-band). Replaces ~668 lines of scoped CSS with `.dp-*`
  classes and component imports. The hardest single change in this
  phase; requires browser preview at 390px and 1280px.
- **`src/pages/index.astro` — kill the deal-rail negative-margin
  bug** by collapsing the rail into the live module. Resolves the
  `index.astro:163-177` paper cut documented in the audit.
- **`src/components/Header.astro` — review** the desktop padding
  scale once the homepage rebuild lands; if `.wrap` standardises
  the 16/24/32 inline scale, header padding inherits it. Otherwise
  no further header changes are needed.
- **Remove `Header.astro` `.wrap` overrides** if the homepage
  rebuild establishes a single `.wrap` rule consumable site-wide.
- **Spot-check `/design-spike/`** under the new global tokens. The
  spike file scopes its own tokens (`--ql-*`) so should be
  unaffected; if any drift appears, reconcile.

**Owner:** technical-lead (build + deploy), section editors do
copy review. Chairman gate at PR review.

**Acceptance:**
- Lighthouse mobile ≥ 95 on the rebuilt homepage.
- `astro check` clean; `npm run build` succeeds.
- 8 sections become 4 visibly; deal rail merges with tracker; gold
  appears nowhere on the homepage.
- All long-form pages (card review, guide, bank hub, airline
  programme, valuations) still render cleanly under the retuned
  `.dp-*` tokens.

**Branch:** `claude/quiet-ledger-rollout`.

### Phase C — Weekly Firecrawl cadence (the data flywheel)

The whole point of the structure-and-design phase: once governance,
structure, and look/feel are up, the Council fills the data layer
weekly via Firecrawl-driven scrapes of bank websites.

Steps in order:

1. **Restore `welcomeBonus` to `SCRAPED_FIELDS`** in
   `scripts/scrape/propose-changes.ts`. Fenced contract change;
   needs explicit Chairman go (Open Question A on STATUS.md).
2. **Per-bank perk-filter regex** — generalise the FAB-specific
   filter at `_normaliser.ts:130` to a per-bank override map keyed
   by `bankSlug`. ENBD will need its own regex for the
   `emiratesnbd.com`, `online.emiratesnbd.com` chrome.
3. **ENBD scraper.** New `scripts/scrape/enbd.ts` and
   `scripts/scrape/banks/enbd.urls.json`. Head of Research produces
   the URL set first via a dossier at
   `.council/research/2026-05/enbd-card-pages-dossier.md`. The
   scraper reuses `_lib.ts` parsers verbatim. Acceptance:
   `FIRECRAWL_API_KEY=skip npm run scrape:enbd` runs; with a real
   key, the 9 pre-seeded ENBD cards in `cards.json` get
   `_provenance: scraped` (or stay `editor-confirmed` where
   already promoted).
4. **Workflow matrixisation.** Replace the hard-coded `npm run
   scrape:fab` line in `.github/workflows/scrape.yml` with a matrix
   job iterating bank slugs. The matrix is fed from a registry at
   `scripts/scrape/banks.registry.json` listing every implemented
   scraper. New banks get added by appending to the registry, not by
   editing the workflow. Failure of one bank does not abort the
   others (`fail-fast: false`).
5. **Rollout order for the remaining 9 banks** (per scrape brief):
   ADCB → Mashreq → HSBC UAE → RAKBANK → Citi UAE → CBD → DIB →
   Standard Chartered UAE → ADIB. One bank per fortnight cadence
   keeps Head of Research within Firecrawl quota.
6. **Cron acceptance.** First scheduled run after ENBD lands; PR
   opens automatically against `main`; never auto-merges; Managing
   Editor reviews the merge with the Fact-Checker on standby.

**Branches:** one per bank (e.g. `claude/scrape-enbd`,
`claude/scrape-adcb`). Workflow matrix and `welcomeBonus` contract
restoration land in their own PRs first.

  explicit Chairman authorisation to provision the account.
- Cloudflare Web Analytics token swap — needs the real token
  committed (Chairman call).
- Comment system (per `BRAND_NOTES.md` §5, recommendation C: none
  at launch).

   compress the 9-bank rollout from ~18 weeks to ~9 weeks?
